# 11-安装部署与用户配置指南

## 1. 安装目标

用户安装 GameSound TUI 后，需要完成三件事：

1. 安装或准备虚拟音频设备；
2. 在 GameSound 中选择真实麦克风、虚拟输出和监听设备；
3. 在语音软件中选择虚拟设备作为麦克风输入。

## 2. Windows 配置流程

### 2.1 安装虚拟音频设备

推荐：VB-CABLE 或 Voicemeeter。

安装后系统中通常出现：

- CABLE Input，作为软件输出设备；
- CABLE Output，作为语音软件输入设备。

### 2.2 GameSound 设置

在 TUI 中：

```text
M 选择真实麦克风，例如 USB Microphone
O 选择虚拟输出设备，例如 CABLE Input
L 选择监听设备，例如 Headphones
```

### 2.3 语音软件设置

在 Discord / QQ / 游戏语音中：

- 输入设备选择：CABLE Output；
- 输出设备选择：用户耳机；
- 关闭自动增益或噪声抑制可按实际效果调整。

## 3. macOS 配置流程

### 3.1 安装虚拟音频设备

推荐：BlackHole。

安装后系统中出现：

- BlackHole 2ch。

### 3.2 GameSound 设置

```text
Mic Input: 用户真实麦克风
Virtual Output: BlackHole 2ch
Monitor: 用户耳机
```

### 3.3 语音软件设置

在 Discord / 微信语音 / 游戏语音中：

- 输入设备选择 BlackHole 2ch；
- 输出设备选择耳机。

## 4. Linux 配置流程

Linux 支持作为后续目标，可基于 PulseAudio 或 PipeWire 创建虚拟 sink/source。

示例思路：

```text
GameSound 输出到虚拟 sink
语音软件输入选择对应 monitor source
```

## 5. 首次启动向导

首次启动时，TUI 应引导用户：

1. 选择真实麦克风；
2. 选择虚拟输出设备；
3. 选择监听设备；
4. 播放测试音；
5. 提示用户打开语音软件选择虚拟输入；
6. 完成配置。

## 6. 常见问题

### 6.1 队友只能听到音效，听不到我说话

原因：语音软件选择了虚拟麦克风，但 GameSound 没有正确采集真实麦克风或未启用混音。

检查：

- GameSound 中 Mic Input 是否选择真实麦克风；
- 麦克风电平是否有变化；
- 麦克风音量是否为 0；
- 是否静音了麦克风。

### 6.2 队友听不到音效

检查：

- GameSound 的 Virtual Output 是否选择虚拟设备；
- 语音软件输入是否选择对应虚拟设备；
- 音效音量是否为 0；
- 快捷键是否触发成功；
- 音频文件是否存在。

### 6.3 我自己听不到音效

检查：

- 是否开启本地监听；
- Monitor 设备是否选择耳机；
- Monitor 音量是否为 0；
- 监听模式是否正确。

### 6.4 有回声

可能原因：

- 监听输出被麦克风重新采集；
- 语音软件监听开启；
- 扬声器外放导致麦克风拾音。

建议：

- 使用耳机；
- 监听模式选择仅音效；
- 关闭语音软件内置监听。

## 7. 卸载与清理

卸载 GameSound 时，应提示是否保留：

- 音频库；
- 配置文件；
- 日志；
- 备份。

虚拟音频设备通常由第三方工具安装，GameSound 不应擅自卸载。
