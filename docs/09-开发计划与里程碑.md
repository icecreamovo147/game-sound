# 09-开发计划与里程碑

## 1. 开发目标

本计划面向完整功能实现，不以 MVP 为最终目标。开发过程可分阶段推进，但每个阶段都应服务于最终完整版本架构，避免临时实现导致后续重构成本过高。

## 2. 阶段一：项目基础架构

目标：建立 Rust workspace 和模块边界。

任务：

- 创建 workspace；
- 建立 gamesound-core；
- 建立 gamesound-storage；
- 建立 gamesound-tui；
- 接入 tracing 日志；
- 建立错误类型；
- 建立 RuntimeCommand / RuntimeEvent；
- 建立基础配置目录。

交付：

- 可启动空 TUI；
- 可读取配置；
- 可写日志。

## 3. 阶段二：TUI 主界面

任务：

- 实现主布局；
- 实现音效列表组件；
- 实现分类栏；
- 实现设备状态栏；
- 实现快捷键提示栏；
- 实现帮助页面；
- 实现基础弹窗框架。

交付：

- 可通过键盘操作 TUI；
- 支持页面切换；
- 支持退出和帮助。

## 4. 阶段三：数据存储与音频库

任务：

- 实现 SQLite 表结构；
- 实现迁移；
- 实现音频添加、编辑、删除；
- 实现分类管理；
- 实现音效方案管理；
- 实现搜索和筛选。

交付：

- 音效库可持久保存；
- TUI 中可管理音效。

## 5. 阶段四：设备枚举与配置

任务：

- 使用 cpal 枚举输入设备；
- 枚举输出设备；
- 实现设备选择弹窗；
- 实现设备配置保存；
- 实现设备失效检测；
- 实现设备测试。

交付：

- 用户可选择麦克风、虚拟输出、本地监听设备。

## 6. 阶段五：音效解码与播放

任务：

- 接入 Symphonia；
- 支持 WAV / MP3 / M4A 等格式；
- 实现播放实例；
- 实现播放、停止、暂停、循环；
- 实现播放状态回传；
- 实现音效音量控制。

交付：

- TUI 中可播放音效；
- 播放状态实时更新。

## 7. 阶段六：麦克风采集与混音

任务：

- 实现麦克风采集；
- 实现采样格式转换；
- 实现重采样和声道转换；
- 实现实时混音；
- 实现限幅；
- 实现麦克风、音效、监听音量控制；
- 实现电平统计。

交付：

- 真实麦克风 + 音效可混音输出。

## 8. 阶段七：虚拟输出与本地监听

任务：

- 将混音结果写入虚拟音频设备；
- 实现本地监听输出；
- 支持监听模式切换；
- 实现输出设备断线恢复；
- 编写平台配置指南。

交付：

- 语音软件可采集混音结果。

## 9. 阶段八：全局快捷键

任务：

- 接入全局快捷键库；
- 实现音效快捷键绑定；
- 实现控制类快捷键；
- 实现快捷键冲突检测；
- 实现运行中重新注册；
- 实现绑定弹窗捕获按键。

交付：

- 用户切到游戏后仍可快捷键播放音效。

## 10. 阶段九：高级音频能力

任务：

- Ducking；
- 噪声门；
- 软限幅；
- 音效淡入淡出；
- 多播放策略；
- 播放队列；
- 互斥组。

交付：

- 完整音频控制能力。

## 11. 阶段十：导入导出与诊断

任务：

- 方案导入导出；
- 配置备份恢复；
- doctor 诊断命令；
- 日志页面；
- 配置损坏恢复；
- 音频文件失效清理。

交付：

- 软件具备长期使用和维护能力。

## 12. 阶段十一：跨平台适配与打包

任务：

- Windows 适配；
- macOS 适配；
- Linux 适配评估；
- 终端兼容性测试；
- 安装包制作；
- 发布说明；
- 用户文档。

交付：

- 可发布的完整版本。

## 13. 阶段十二：后续演进

任务：

- Tauri GUI 客户端；
- 自研虚拟麦克风驱动；
- 音频剪辑；
- AI TTS 生成；
- 语音包市场；
- 云同步。
