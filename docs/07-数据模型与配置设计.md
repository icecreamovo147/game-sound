# 07-数据模型与配置设计

## 1. 数据存储目标

系统需要保存音频库、分类、音效方案、快捷键、播放统计、设备配置和用户偏好。完整版本建议使用 SQLite 保存结构化数据，使用 TOML 保存全局配置。

## 2. 配置目录

建议目录：

```text
~/.gamesound/
├── config.toml
├── gamesound.db
├── sounds/
├── profiles/
├── backups/
└── logs/
```

## 3. SQLite 表设计

### 3.1 sounds

```sql
CREATE TABLE sounds (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    file_path TEXT NOT NULL,
    category_id INTEGER,
    profile_id INTEGER,
    volume REAL NOT NULL DEFAULT 0.8,
    playback_mode TEXT NOT NULL DEFAULT 'once',
    loop_enabled INTEGER NOT NULL DEFAULT 0,
    favorite INTEGER NOT NULL DEFAULT 0,
    tags TEXT,
    note TEXT,
    duration_ms INTEGER,
    sample_rate INTEGER,
    channels INTEGER,
    file_size INTEGER,
    sort_order INTEGER NOT NULL DEFAULT 0,
    play_count INTEGER NOT NULL DEFAULT 0,
    last_played_at TEXT,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);
```

### 3.2 categories

```sql
CREATE TABLE categories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    profile_id INTEGER,
    sort_order INTEGER NOT NULL DEFAULT 0,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);
```

### 3.3 profiles

```sql
CREATE TABLE profiles (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    description TEXT,
    is_active INTEGER NOT NULL DEFAULT 0,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);
```

### 3.4 hotkeys

```sql
CREATE TABLE hotkeys (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    profile_id INTEGER,
    sound_id INTEGER,
    action TEXT NOT NULL,
    hotkey TEXT NOT NULL,
    enabled INTEGER NOT NULL DEFAULT 1,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);
```

### 3.5 playback_history

```sql
CREATE TABLE playback_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    sound_id INTEGER NOT NULL,
    triggered_by TEXT NOT NULL,
    played_at TEXT NOT NULL
);
```

### 3.6 app_settings

```sql
CREATE TABLE app_settings (
    key TEXT PRIMARY KEY,
    value TEXT NOT NULL,
    updated_at TEXT NOT NULL
);
```

## 4. TOML 配置设计

`config.toml` 示例：

```toml
[app]
first_run = false
active_profile = "default"
log_level = "info"
auto_save = true

[tui]
theme = "default"
tick_rate_ms = 100
show_level_meter = true
confirm_on_delete = true

[audio]
sample_rate = 48000
channels = 2
buffer_size = 512
playback_mode = "interrupt"

[audio.devices]
mic = "USB Microphone"
output = "BlackHole 2ch"
monitor = "Headphones"

[volume]
mic = 0.9
sfx = 0.8
monitor = 0.6

[monitor]
enabled = true
mode = "sfx_only"

[ducking]
enabled = true
ratio = 0.4
attack_ms = 50
release_ms = 300
release_delay_ms = 200

[hotkeys]
enabled = true
stop_all = "ctrl+alt+s"
toggle_mic = "ctrl+alt+m"
toggle_sfx = "ctrl+alt+x"
```

## 5. 配置迁移

每次数据库结构变化时应记录迁移版本。

```sql
CREATE TABLE schema_migrations (
    version INTEGER PRIMARY KEY,
    applied_at TEXT NOT NULL
);
```

启动时检查当前版本，并按顺序执行迁移。

## 6. 备份策略

完整版本应支持：

- 每次重大配置变更前自动备份；
- 每日首次启动自动备份；
- 用户手动备份；
- 备份文件保留数量限制；
- 从备份恢复。

备份目录：

```text
~/.gamesound/backups/YYYYMMDD-HHMMSS/
```

## 7. 音频文件管理策略

导入音频时支持两种模式：

1. 引用原路径；
2. 复制到 `~/.gamesound/sounds/`。

推荐默认复制到软件目录，避免原文件移动后失效。

## 8. 数据一致性

需要处理：

- 音频文件路径失效；
- 分类删除后的音效迁移；
- 方案删除后的音效处理；
- 快捷键绑定到不存在音效；
- 设备名称变化导致配置失效。
