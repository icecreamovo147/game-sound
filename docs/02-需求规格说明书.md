# 02-需求规格说明书

## 1. 总体需求

GameSound TUI 应提供一个终端可视化界面，允许用户管理音频素材、配置音频设备、设置全局快捷键，并在运行期间将真实麦克风输入与音效播放音频进行实时混音，输出到虚拟麦克风设备。

完整版本应支持专业级可用体验，包括音频库管理、分类管理、播放策略、设备配置、混音控制、快捷键触发、本地监听、状态监控、配置备份和异常恢复。

## 2. 功能需求

### 2.1 音频库管理

#### 2.1.1 导入音频

系统应支持导入本地音频文件。

支持格式：

- WAV
- MP3
- M4A
- OGG
- FLAC
- AAC

导入方式：

- 单文件导入；
- 多文件批量导入；
- 文件夹扫描导入；
- 拖入路径导入，适用于终端中输入路径；
- 后续支持导入配置包。

导入时应记录：

- 音频名称；
- 文件路径；
- 文件格式；
- 时长；
- 采样率；
- 声道数；
- 文件大小；
- 创建时间；
- 最后修改时间。

#### 2.1.2 音频编辑

系统应支持编辑音频条目属性：

- 名称；
- 分类；
- 标签；
- 快捷键；
- 默认音量；
- 播放模式；
- 是否循环；
- 是否收藏；
- 备注；
- 排序权重。

#### 2.1.3 音频删除

系统应支持删除音频条目。

删除时应支持两种策略：

- 仅从音频库移除记录，不删除源文件；
- 同时删除复制到软件音频目录中的文件。

删除前需二次确认。

#### 2.1.4 音频搜索与筛选

系统应支持：

- 按名称搜索；
- 按分类筛选；
- 按标签筛选；
- 按快捷键搜索；
- 按收藏筛选；
- 按播放次数排序；
- 按最近使用排序。

### 2.2 分类与音效方案

#### 2.2.1 分类管理

系统应支持：

- 新建分类；
- 重命名分类；
- 删除分类；
- 分类排序；
- 分类内音效数量显示。

#### 2.2.2 音效方案

完整版本应支持多个音效方案，例如：

- LOL 方案；
- 瓦罗兰特方案；
- CS2 方案；
- 抽象语音包方案；
- 朋友专属语音包方案。

每个方案可拥有独立的：

- 音效列表；
- 分类；
- 快捷键绑定；
- 播放策略；
- 音量设置。

### 2.3 播放控制

系统应支持以下播放操作：

- 播放选中音效；
- 停止选中音效；
- 停止全部音效；
- 暂停音效；
- 恢复音效；
- 循环播放；
- 重新播放；
- 音效淡入；
- 音效淡出；
- 播放进度显示。

### 2.4 播放策略

系统应支持多种播放模式：

#### 2.4.1 打断模式

触发新音效时，停止当前正在播放的音效，立即播放新音效。

#### 2.4.2 叠加模式

多个音效可以同时播放，最终在混音器中叠加。

#### 2.4.3 队列模式

新触发音效进入队列，等待当前音效播放完成后继续播放。

#### 2.4.4 同组互斥模式

同一分类或同一互斥组中的音效互相打断，不同组之间可以叠加。

### 2.5 快捷键需求

系统应支持全局快捷键，即应用不在焦点时仍能触发。

快捷键功能包括：

- 播放指定音效；
- 停止全部音效；
- 静音麦克风；
- 静音音效；
- 开关本地监听；
- 切换音效方案；
- 增加/降低麦克风音量；
- 增加/降低音效音量；
- 增加/降低监听音量。

系统应支持快捷键冲突检测：

- 音效之间快捷键冲突；
- 系统级快捷键可能冲突；
- TUI 操作键与全局快捷键区分。

### 2.6 麦克风输入需求

系统应支持选择真实麦克风输入设备。

功能要求：

- 枚举系统输入设备；
- 选择默认输入设备；
- 切换指定输入设备；
- 检测设备失效；
- 设备失效后提示用户重新选择；
- 显示实时麦克风电平；
- 支持麦克风静音；
- 支持麦克风音量调节；
- 支持基础增益控制。

增强需求：

- 噪声门；
- 自动增益控制；
- 简单降噪；
- 麦克风监听测试。

### 2.7 音效混音需求

系统应将真实麦克风输入与软件音效进行混音。

混音器应支持：

- 多路音效叠加；
- 麦克风音量独立控制；
- 音效总音量独立控制；
- 单个音效音量控制；
- 防爆音限幅；
- 输出电平统计；
- 声道转换；
- 采样率转换；
- 播放延迟控制。

### 2.8 Ducking 需求

系统应支持播放音效时自动降低麦克风音量。

配置项：

- 是否启用 Ducking；
- 麦克风降低比例；
- 攻击时间；
- 释放时间；
- 恢复延迟。

示例：

- 音效播放期间，麦克风音量降至 40%；
- 音效结束 300ms 后恢复到原始音量。

### 2.9 虚拟麦克风输出需求

由于真实物理麦克风无法直接接收软件内部音频流，系统应输出到虚拟音频设备。

功能要求：

- 枚举系统输出设备；
- 选择虚拟音频输出设备；
- 标记疑似虚拟音频设备；
- 输出混音音频流；
- 检测输出设备失效；
- 支持输出测试。

平台建议：

- Windows：VB-CABLE、Voicemeeter、Virtual Audio Cable；
- macOS：BlackHole、Loopback；
- Linux：PulseAudio / PipeWire 虚拟 sink/source。

### 2.10 本地监听需求

用户可选择是否在本地耳机中听到音效或混音结果。

功能要求：

- 开启/关闭监听；
- 选择监听输出设备；
- 监听音量独立控制；
- 可选择监听内容：仅音效、仅麦克风、完整混音；
- 避免监听回流导致回声。

### 2.11 TUI 界面需求

系统应提供终端可视化界面，支持：

- 音效列表展示；
- 分类展示；
- 设备状态展示；
- 电平条展示；
- 播放状态展示；
- 快捷键提示；
- 弹窗操作；
- 搜索输入；
- 配置页面；
- 帮助页面；
- 错误提示。

### 2.12 配置持久化

系统应持久保存：

- 音频库；
- 分类；
- 音效方案；
- 快捷键绑定；
- 设备选择；
- 音量设置；
- 播放策略；
- TUI 偏好；
- 日志级别。

### 2.13 导入导出与备份

完整版本应支持：

- 导出音效方案；
- 导入音效方案；
- 备份配置；
- 恢复配置；
- 清理无效音频路径；
- 迁移配置目录。

### 2.14 日志与诊断

系统应提供日志与诊断能力：

- 启动日志；
- 设备枚举日志；
- 快捷键注册日志；
- 音频流错误日志；
- 播放事件日志；
- 崩溃日志；
- 诊断命令或诊断页面。

## 3. 非功能需求

### 3.1 性能需求

- 音频触发延迟应尽可能低；
- 快捷键触发到音效开始播放的延迟建议小于 100ms；
- 混音处理不应造成明显卡顿；
- TUI 刷新不应影响音频线程；
- 音频线程应高优先级、低阻塞。

### 3.2 稳定性需求

- 音频设备断开时不应导致程序崩溃；
- 音频文件不存在时应提示错误；
- 快捷键注册失败时应给出原因；
- 混音线程异常时应尝试恢复或安全退出；
- 配置损坏时应支持恢复默认配置。

### 3.3 兼容性需求

目标平台：

- Windows 10 / 11；
- macOS 13+；
- Linux 可作为后续支持。

终端兼容：

- Windows Terminal；
- PowerShell；
- macOS Terminal；
- iTerm2；
- 常见 Linux Terminal。

### 3.4 可维护性需求

- 音频核心与 TUI 解耦；
- 运行时事件与 UI 事件解耦；
- 配置层独立；
- 支持后续 GUI 复用 core；
- 关键模块需要单元测试。

### 3.5 用户体验需求

- 常用操作不应依赖复杂命令；
- TUI 快捷键应清晰显示；
- 设备配置失败应有明确引导；
- 首次使用应提供初始化向导；
- 错误提示应面向普通用户可理解。

## 4. 异常场景

| 场景 | 处理方式 |
|---|---|
| 音频文件不存在 | 标记为失效，播放时提示 |
| 音频格式不支持 | 导入失败并提示格式 |
| 麦克风设备断开 | 自动静音麦克风输入并提示重新选择 |
| 输出设备断开 | 停止输出并提示重新选择 |
| 快捷键冲突 | 阻止绑定或提示覆盖 |
| 配置文件损坏 | 备份损坏文件并生成默认配置 |
| 数据库损坏 | 尝试修复，失败则提示恢复备份 |
| 音频线程崩溃 | 记录日志并尝试重启音频引擎 |
| 终端窗口尺寸过小 | 显示简化布局或提示调整窗口 |
