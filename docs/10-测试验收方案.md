# 10-测试验收方案

## 1. 测试目标

验证 GameSound TUI 在功能、音频稳定性、快捷键、设备兼容性、TUI 交互、配置持久化和跨平台方面满足完整版本要求。

## 2. 功能测试

### 2.1 音频库测试

| 用例 | 预期结果 |
|---|---|
| 添加 MP3 文件 | 成功导入并显示在列表 |
| 添加 WAV 文件 | 成功导入并可播放 |
| 添加不存在文件 | 提示路径无效 |
| 删除音效 | 列表移除并保存 |
| 修改音效名称 | 列表实时更新 |
| 搜索关键词 | 正确筛选音效 |
| 切换分类 | 显示对应分类音效 |

### 2.2 播放测试

| 用例 | 预期结果 |
|---|---|
| Enter 播放选中音效 | 音效开始播放 |
| 停止全部 | 所有音效停止 |
| 循环播放 | 音效结束后继续播放 |
| 打断模式 | 新音效停止旧音效 |
| 叠加模式 | 多音效同时播放 |
| 队列模式 | 音效按顺序播放 |

### 2.3 快捷键测试

| 用例 | 预期结果 |
|---|---|
| 绑定 Ctrl+1 | 绑定成功 |
| 重复绑定 | 提示冲突 |
| 游戏窗口中按快捷键 | 音效正常触发 |
| 停止快捷键 | 所有音效停止 |
| 修改快捷键 | 旧快捷键失效，新快捷键生效 |

## 3. 音频链路测试

### 3.1 麦克风采集

- 麦克风电平随说话变化；
- 静音后无麦克风输出；
- 音量调整生效；
- 麦克风断开后提示错误。

### 3.2 混音输出

验收标准：

- 队友能够同时听到用户说话和音效；
- 音效不明显爆音；
- 音效播放延迟可接受；
- 长时间运行无明显卡顿；
- 本地监听不产生明显回声。

### 3.3 Ducking

- 音效播放时麦克风音量自动降低；
- 音效结束后麦克风音量恢复；
- 关闭 Ducking 后不再触发。

## 4. TUI 测试

| 用例 | 预期结果 |
|---|---|
| 终端启动 | 正常进入主界面 |
| 窗口缩放 | 布局正常适配 |
| 上下键移动 | 选中项变化 |
| Tab 切换 | 焦点区域变化 |
| 弹窗打开关闭 | 状态正确恢复 |
| 帮助页 | 显示完整快捷键说明 |
| 日志页 | 正确显示日志 |

## 5. 配置测试

- 修改设备后重启仍保留；
- 修改音量后重启仍保留；
- 音效列表持久化；
- 快捷键持久化；
- 配置损坏时可恢复默认；
- 数据库迁移正常。

## 7. 性能测试

指标建议：

| 指标 | 目标 |
|---|---|
| 快捷键触发延迟 | 小于 100ms |
| TUI 刷新间隔 | 50ms - 100ms |
| CPU 占用 | 空闲时较低 |
| 内存占用 | 音频库规模合理时稳定 |
| 长时间运行 | 2 小时以上无崩溃 |

## 8. 稳定性测试

- 播放过程中断开麦克风；
- 播放过程中断开耳机；
- 删除正在播放的音频；
- 快速连续触发快捷键；
- 同时播放多个音效；
- 切换方案时仍有音效播放；
- 终端窗口尺寸极小。

## 9. 验收标准

完整版本验收应满足：

1. 可稳定启动 TUI；
2. 可管理音效库；
3. 可配置真实麦克风和虚拟输出；
4. 可将麦克风与音效混音；
5. 语音软件可采集混音结果；
6. 全局快捷键可在游戏中触发；
7. 配置可持久保存；
8. 异常场景不崩溃；
9. 日志和诊断可辅助定位问题；
10. Windows 和 macOS 至少一个平台达到可发布质量。
