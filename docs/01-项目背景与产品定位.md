# 01-项目背景与产品定位

## 1. 项目背景

在多人游戏开黑、语音聊天、直播互动等场景中，用户常常希望在语音频道中快速播放一些有趣的音效、语音包、BGM 片段或提示音，以增强娱乐性和互动效果。传统方式通常包括外放音频让麦克风拾音、手动切换播放器、使用复杂的虚拟声卡软件或依赖直播软件混音。这些方式存在操作繁琐、音质差、延迟高、配置复杂、不适合游戏中快速触发等问题。

本项目旨在开发一款以终端 TUI 形式运行的游戏音效板工具。用户通过终端启动程序后，可以在 TUI 界面中管理音效、配置设备、设置快捷键和查看运行状态。程序运行期间会持续监听全局快捷键，用户即使切换到游戏窗口，也可以通过快捷键触发音效播放。

与普通音效播放器不同，本项目的核心能力是：将真实麦克风输入与软件播放音效进行实时混音，并输出到虚拟麦克风设备，使语音软件能够同时采集用户说话声和音效声。

## 2. 产品名称

暂定名称：**GameSound TUI**

备选名称：

- GameSoundPad
- MicSoundboard
- VoiceBoard TUI
- 开黑音效板
- 麦克风音效助手

## 3. 产品定位

GameSound TUI 是一款面向游戏开黑和语音娱乐场景的终端音效板工具，提供音频库管理、全局快捷键触发、麦克风实时混音、虚拟麦克风输出、本地监听和设备配置能力。

一句话描述：

> 一款运行在终端中的游戏语音音效板，支持快捷键播放自定义音效，并将真实麦克风与音效混音后输出到虚拟麦克风设备。

## 4. 目标用户

### 4.1 主要用户

- 经常与朋友开黑、语音聊天的游戏玩家；
- 想在语音频道播放搞笑音效、语音包、梗图配音、BGM 片段的用户；
- 熟悉终端操作，希望用轻量工具完成音频混音和快捷键播放的用户；
- 不想使用复杂 GUI 软件或直播软件，但需要稳定音频路由能力的用户。

### 4.2 次要用户

- 主播、视频创作者、播客用户；
- 需要低成本音效触发器的桌面端用户；
- 希望后续基于 core 模块扩展 GUI 客户端的开发者。

## 5. 核心场景

### 场景一：游戏中快捷键播放音效

用户启动 GameSound TUI 后进入游戏，按下绑定好的全局快捷键，例如 `Ctrl + 1`，软件立即播放对应音效。队友在语音频道中能够听到音效，同时仍然可以听到用户的真实说话声。

### 场景二：管理语音包和音效库

用户在 TUI 中导入本地音频文件，为音频设置名称、分类、音量、快捷键、播放模式等属性。音频可以按分类筛选、搜索、收藏和排序。

### 场景三：配置麦克风混音链路

用户选择真实麦克风输入设备、虚拟麦克风输出设备和本地监听设备。软件采集真实麦克风音频，与音效音频混合后输出到虚拟音频设备。语音软件选择该虚拟设备作为输入源。

### 场景四：终端长期运行

用户在终端执行 `gamesound` 或 `gamesound start` 后进入 TUI。程序长期运行，终端界面显示运行状态、音效列表、设备状态、音量和快捷键提示。

## 6. 产品边界

### 6.1 本项目包含

- TUI 音效板界面；
- 音频库管理；
- 全局快捷键触发；
- 麦克风采集；
- 音效解码和播放；
- 实时混音；
- 虚拟音频设备输出；
- 本地监听；
- 设备检测与配置；
- 配置持久化；
- 日志与错误提示；
- 跨平台基础支持。

### 6.2 本项目不在第一阶段内置但可长期规划

虽然本文档以完整功能为目标，但需明确部分能力的技术边界：

- 自研虚拟麦克风驱动属于高级阶段；
- 音频剪辑、AI 语音生成、云端语音包市场属于增强功能；
- 图形 GUI 客户端可以作为后续客户端形态；
- 移动端支持不属于当前项目范围。

## 7. 完整产品目标

完整版本应实现：

1. 终端 TUI 可视化管理音效；
2. 支持多分类、多音效方案、多快捷键绑定；
3. 支持真实麦克风与音效实时混音；
4. 支持输出到第三方虚拟音频设备；
5. 支持本地耳机监听；
6. 支持全局快捷键后台触发；
7. 支持音量、电平、播放状态实时显示；
8. 支持音效队列、叠加、打断、循环等播放策略；
9. 支持配置保存、导入导出和备份；
10. 支持后续 GUI 和自研虚拟麦克风扩展。
